**Version:** 1.1

## Overview

This document consolidates all user testing requirements for the v1.1 release of the Meal Planner application. It includes new features, enhancements, and regression testing requirements.

## New Features in v1.1

### 1. Browse Recipes Feature (Issue #67)
**Priority:** High  
**Status:** Implementation Complete

A new feature that allows users to search and browse 360,000+ recipes from Spoonacular API and add them to their recipe box.

**Test Coverage:**
- Automated E2E tests: `frontend/e2e/tests/recipes/browse-recipes.spec.ts`
- Manual testing required: Yes

**Related Documentation:**
- Implementation details: See GitHub Issue #67
- Sub-issues: #68 (MVP), #69 (Filters), #70 (Polish & Testing)

**Tester Feedback**
- It is a little confusing to have "search recipes" and "browse recipes" in the left nav. Condense into a singular "recipes" then have a facility within for the user to navigate between their own personal/family recipes and a broader recipe search of spoonacular etc - while retaining the recipe import function.  
- Add a mechanism to allow the user to upload images/pdf/documents of recipes which then get added to their own personal/family recipes
- Current recipes look to be sample data with no ingredients which makes it very hard to test. Suggest matching recipes in the test db with recipes that contain ingredient/instruction details from spoonacular. If there's no match then just delete the recipe from the test database
- Spoonacular does not seem to be working - search queries return no results.

3. **Recipe Details:** Not all recipes have complete nutritional information

## Automated Test Execution

### E2E Tests
```bash
# Run all E2E tests
cd frontend
pnpm test:e2e

# Run Browse Recipes tests only
pnpm test:e2e browse-recipes

# Run with UI
pnpm test:e2e:ui

# Run in headed mode (see browser)
pnpm test:e2e:headed
```

### Test Reports
- Test results: `frontend/test-results/`
- HTML report: `pnpm test:e2e:report`
